Overview

This API acts as a clean, standardized gateway to the TripAdvisor Content API. Unlike scraping-based alternatives, every response comes directly from TripAdvisor's official infrastructure — meaning no breakage, no blocks, no stale data.

ℹ️
This is a BYOK (Bring Your Own Key) API. You need a free TripAdvisor Partner API key to use it. See the Quick Start section to get yours in under 5 minutes.
Base URL https://your-api.p.rapidapi.com/v1

All requests use HTTPS and return JSON. The TripAdvisor key is passed in every request via the X-TripAdvisor-Key header.

02

Authentication

🔐 RapidAPI Key

Authenticate all requests with your RapidAPI subscription key in the header.

HEADER
X-RapidAPI-Key
VALUE
YOUR_RAPIDAPI_KEY

🗝️ TripAdvisor Partner Key (BYOK)

Pass your personal TripAdvisor Content API key in every request. It's free to obtain — see Quick Start below.

HEADER
X-TripAdvisor-Key
VALUE
YOUR_TA_PARTNER_KEY
03

Quick Start

1

Create a TripAdvisor Developer Account

Go to tripadvisor.com/developers and sign up for free. No credit card required for the Partner API.

2

Generate your API Key

In the developer dashboard, navigate to Credentials and create a new key. Copy it — you'll need it in every request.

3

Subscribe on RapidAPI

Choose a plan that fits your usage. The Free plan includes 500 requests/month — enough to test everything.

4

Make your first request

Pass both keys in the headers and you're live. See the example below.

curl "https://your-api.p.rapidapi.com/v1/search?searchQuery=Copacabana&category=restaurants" \ -H "X-RapidAPI-Key: YOUR_RAPIDAPI_KEY" \ -H "X-TripAdvisor-Key: YOUR_TA_KEY"
import requests headers = { "X-RapidAPI-Key": "YOUR_RAPIDAPI_KEY", "X-TripAdvisor-Key": "YOUR_TA_KEY" } response = requests.get( "https://your-api.p.rapidapi.com/v1/search", headers=headers, params={"searchQuery": "Copacabana", "category": "restaurants"} ) print(response.json())
const res = await fetch( "https://your-api.p.rapidapi.com/v1/search?searchQuery=Copacabana&category=restaurants", { headers: { "X-RapidAPI-Key": "YOUR_RAPIDAPI_KEY", "X-TripAdvisor-Key": "YOUR_TA_KEY" } } ); const data = await res.json(); console.log(data);

Endpoints

GET
/v1/search
Returns up to 10 locations matching a text query. Supports filtering by category, address, phone, and geographic radius.
native
▼
Query Parameters
ParameterTypeDescription
searchQuery*stringText to search — name, city, or address of the location
categorystringFilter by type: hotels, attractions, restaurants, geos
phonestringPhone number to narrow results (without the + prefix)
addressstringAddress string to filter results
latLongstringLat/Long pair to bias results geographically — e.g. -22.9068,-43.1729
radiusnumberSearch radius from latLong point
radiusUnitstringUnit for radius: km, mi, m
languagestringResponse language code. Default: en. Supports 45 languages including pt (Portuguese)

Error Handling

All errors follow a standard structure with a human-readable message and an error code.

4xx / 5xx Error Response Structure
{ "error": { "code": "INVALID_TA_KEY", "message": "Your TripAdvisor API key is invalid or expired.", "docs": "https://tripadvisor.com/developers" } }
400
BAD_REQUEST — Missing required parameter (e.g. searchQuery or latLong)
401
INVALID_TA_KEY — TripAdvisor key missing, invalid, or expired
401
INVALID_RAPIDAPI_KEY — RapidAPI subscription key is invalid
429
TA_RATE_LIMIT — Your TripAdvisor key exceeded its rate limit (50 req/s). Retry after 1 second.
429
PLAN_LIMIT — Your RapidAPI plan monthly request limit reached
500
UPSTREAM_ERROR — TripAdvisor API returned an unexpected error. Usually transient — retry with exponential backoff.
